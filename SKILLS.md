# Repository Skills

## DevSecOps Platform Template Work

Use this workflow when updating the repository:

1. Review `docs/assessment/current-state-assessment.md`.
2. Check `templates/` before creating new starter content.
3. Put reusable platform functionality in `modules/` or `scripts/`.
4. Put client-facing policy in `policies/`.
5. Put step-by-step operational docs in `docs/implementation/` or template runbooks.
6. Validate with safe local checks from `Makefile`.

## Public Safety

All examples must remain safe to publish. Real operational values belong in private implementation repositories.
