# Agent Instructions

## Purpose

This repository is a public-safe DevSecOps platform blueprint and starter repository system.

## Working Rules

- Inspect existing docs, templates, modules, scripts, policies, and examples before changing structure.
- Preserve useful content unless it is clearly duplicated, broken, or obsolete.
- Keep public examples sanitized. Do not add real secrets, client names, account IDs, hostnames, IP addresses, kubeconfigs, Terraform state, or incident evidence.
- Support both VM/Docker Compose clients and Kubernetes-based environments.
- Keep CI in app repo templates and CD in deployment repo templates.
- Keep reusable platform logic in `modules/`, `scripts/`, `policies/`, and `templates/`.
- Prefer concrete examples over abstract placeholders, but use safe placeholder values such as `example.com`.
