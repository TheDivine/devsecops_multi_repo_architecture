# Support Boundaries

## Purpose

Define where ownership starts and stops across repositories and teams.

## Application Support

Application teams respond to application behavior, code defects, dependency updates, and service-level metrics.

## Deployment Support

Deployment owners respond to manifest errors, environment promotion issues, GitOps drift, and rollback execution.

## Infrastructure Support

Infrastructure owners respond to provisioning failures, capacity constraints, network controls, and host-level configuration.

## Platform Support

Platform owners respond to shared controllers, GitOps tooling, policy engines, registry standards, and platform add-ons.

## Security Support

Security owners respond to policy exceptions, hardening baselines, scanner configuration, and detection requirements.

## Monitoring Support

Monitoring owners respond to telemetry gaps, alert routing, dashboard standards, and alert quality.
