# Contributing

Contributions should keep this repository generic, reusable, and safe for public review.

## Contribution Rules

- Prefer documentation, templates, checklists, diagrams, and examples.
- Use placeholder values only.
- Do not add real credentials, tokens, domains, IP addresses, customer names, or private infrastructure details.
- Do not add commands that deploy, mutate infrastructure, or change external systems.
- Keep examples vendor-neutral where possible and call out assumptions when a tool-specific pattern is shown.

## Review Checklist

- The change preserves repository separation between app, deployment, infrastructure, platform, security, monitoring, and docs concerns.
- Any example configuration is clearly non-production and uses placeholders.
- Documentation explains ownership, access boundaries, and rollback or recovery expectations where relevant.
- New files are linked from an appropriate README or index document when useful.
- Safety checks have been run locally before publishing.
