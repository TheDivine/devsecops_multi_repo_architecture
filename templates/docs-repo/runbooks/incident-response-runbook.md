# Incident Response Runbook

## First Response

1. Confirm affected environment and service.
2. Check health, logs, metrics, and recent deployment changes.
3. Decide whether rollback is required.
4. Escalate using the private on-call system.
5. Preserve evidence in approved private locations.

## After Action

- record timeline
- identify root cause
- create corrective actions
- update runbooks and alerts
- review access or security exceptions if relevant
