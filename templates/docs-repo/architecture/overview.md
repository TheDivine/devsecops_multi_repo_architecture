# Architecture Overview

## Purpose

Describe the client or product architecture at a level that is useful for onboarding, operations, and review.

## Required Sections

- business/system purpose
- repository map
- runtime model: VM/Docker Compose, Kubernetes, or both
- ingress model
- data stores
- backup ownership
- monitoring ownership
- security boundaries

Keep private IP addresses, credentials, and customer-sensitive topology in private systems only.
